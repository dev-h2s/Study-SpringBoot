REST
	Representational State Transfer
	(전송에 있어서 무언가를 대표하는 상태) => url을 봤을때 어떤 데이터인지 알수 있게 설계하는 방식을 REST라 부른다
	자원, 행위, 표현으로 구성되어 있다.
	url => ip 포함
	uri => ip 제외
	REST 설계 방식 rui에 자원, 행위, 표현 세가지가 다 담기는것				
	     			 1/ update/ board
	URI만으로도 데이터 및 CRUD의 상태를 이해 할 수 있도록 설계하는 *규칙.
	

	1.소문자로 작성한다.
		대문자로 작성시 문제가 발생할 수 있기 때문에 소문자로 작성한다.
	2. 언더바 대신 하이픈을 사용한다.
		가독성을 높이기 위해서 하이픈으로 구분하는 것이 좋다.
	3. URL 마지막에 슬래시 작성하지 않는다.
		마지막에 작성하는 슬래시는 의미가 없다.
	4. 계층 관계 표현 시 슬래시 구분자로 사용한다.
		계층 관계(포함 관계)에서는 슬래시로 구분해준다.
	5. 동사는 작성하면 안된다.
	
	6. 파일 확장자는 통합하지 않는다.
		행위는 URI로 표현하지 않고 Header의 COntent-Type을 사용하여
		body의 내용을 처리하도록 설계한다.
